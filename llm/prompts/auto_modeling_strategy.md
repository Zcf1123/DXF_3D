# 自动建模策略

这些知识用于 LLM 直接编写 FreeCAD 代码的路线。

## 目标

根据三视图 JSON 摘要和图片摘要，直接生成 FreeCAD Python 建模脚本。

## 推荐推理顺序

- 先根据视图布局、拟合曲线、闭合轮廓、隐藏线和可选用户意图识别主要零件类型。
- 先判断零件属于基础体、板状件、轴/套筒、盘/法兰、支座/支架、壳体、齿形盘、标准件或组合件；若无法确定，优先选择能解释最多投影的保守主体。
- 先构造正实体：圆柱、盒体、拉伸轮廓，以及需要融合的加性组件。
- 正实体融合后再做切除：圆孔、长圆孔、矩形/异形切除和盲切。
- 用 TOP 和 LEFT 判断局部深度与偏移；不要默认把每个 FRONT 组件都拉满总深度。
- 拟合曲线优先使用稳定的 FreeCAD 基元：`Part.makeCylinder`、`Part.Circle`、`Part.Arc`、`Part.Face`、`cut` 和 `fuse`。

## 通用特征落地策略

- 基础实体：圆柱、圆锥、球、盒体、圆环等优先用 `Part.make*` 稳定基元；复杂闭合轮廓先构造 `Part.Wire` / `Part.Face`，再沿正确轴拉伸。
- 孔和槽：优先使用 `hole_hints`；切除体必须是有厚度的 solid，且完全穿透目标实体或按 JSON 给出的盲切深度切除。
- 阶梯轴和多段圆柱：按轴向分段逐段建圆柱并 `fuse`，不要用单个长圆柱吞掉可见端面分界。
- 套筒和衬套：先建外圆柱，再切同轴内孔；不要把内圆忽略成实心件。
- 盘类、法兰和垫片：TOP 的外轮廓通常是 footprint，FRONT/LEFT 给厚度；先建外形，再切中心孔和明确画出的孔。
- 齿轮或带齿圆盘：TOP 齿形轮廓是真实外轮廓，必须沿 Z 拉伸；FRONT/LEFT 主要校验厚度。
- 支座、支架、箱体和壳体：拆成主板、立板、凸台、筋板、型腔和孔槽等可验证实体；缺少壁厚证据时不要凭经验挖空。
- 筋板、加强筋、阵列孔、沉孔、螺纹和钣金折弯只在三视图或结构化 hint 明确支持时建模；否则保守省略或简化为主孔/主实体。

## 禁止凭经验补造

- 不要因为机械常识自动添加未画出的圆角、倒角、螺纹、沉孔、筋板、阵列孔或折弯圆角。
- 不要用“图纸未标注，按常规取值”这类经验值替代 `dimension_constraints` 或 `projected_views` 中的尺寸。
- 不要为了让脚本更简单，把不同局部厚度、不同半径圆柱段或明确分开的组件合并成一个统一实体。

## 输出约束

- 只输出完整 Python 代码。
- 创建最终 `Part::Feature` 对象，名称必须为 `Result`。
- 把最终 solid 赋给 `Result.Shape`。
- 调用 `doc.recompute()` 和 `doc.saveAs(FCSTD_PATH)`。
- 不要使用 shell、网络、文件删除、`eval` 或 `exec`。
